# 每日结余统计功能 - 前端部署说明

## 安装依赖

在前端项目根目录执行：

```bash
cd personal-ledger-frontend
npm install
```

这将自动安装新增的 ECharts 依赖。

## 功能说明

### 页面位置

- 路由路径：`/daily-balance`
- 菜单位置：左侧导航栏 → 每日结余

### 功能特性

1. **折线图展示**
   - 使用 ECharts 绘制每日结余折线图
   - 支持平滑曲线和区域填充
   - 正数（盈余）显示绿色，负数（亏损）显示红色

2. **月份切换**
   - 默认显示当前月份数据
   - 支持通过日期选择器切换月份
   - 格式：YYYY年MM月

3. **统计汇总**
   - 月度总结余：当月所有日期的结余总和
   - 平均每日结余：月度总结余 / 天数
   - 最大单日结余：显示最大结余值及对应日期

4. **响应式设计**
   - 图表自动适应窗口大小
   - 支持窗口缩放时自动调整

5. **空数据处理**
   - 无数据时显示友好提示
   - 自动隐藏统计汇总区域

## 使用方法

### 启动项目

```bash
npm run dev
```

### 访问页面

浏览器访问：`http://localhost:3000/daily-balance`

### 操作步骤

1. 点击左侧导航栏的"每日结余"菜单
2. 页面默认显示当前月份的结余数据
3. 点击右上角的日期选择器可切换月份
4. 查看折线图和统计汇总数据

## 技术实现

### 图表配置

```javascript
{
  tooltip: {
    trigger: 'axis',
    formatter: 自定义格式化（显示颜色和金额）
  },
  xAxis: {
    type: 'category',
    data: 日期列表（MM-DD格式）
  },
  yAxis: {
    type: 'value',
    name: '结余（元）'
  },
  series: [{
    type: 'line',
    smooth: true,  // 平滑曲线
    itemStyle: {
      color: 根据正负值动态设置颜色
    },
    areaStyle: {
      color: 渐变填充
    }
  }]
}
```

### 数据处理

- 日期格式转换：`YYYY-MM-DD` → `MM-DD`
- 结余计算：后端返回（支出 - 收入）
- 统计计算：前端基于返回数据计算汇总值

## 注意事项

1. **依赖安装**：首次使用需要执行 `npm install` 安装 ECharts
2. **后端接口**：确保后端服务已启动且接口可访问
3. **数据范围**：只统计 `include_in_statistics = '1'` 的账单
4. **图表渲染**：需要等待 DOM 渲染完成后再初始化图表

## 常见问题

### Q1: 图表不显示？

**A**: 检查以下几点：
- 是否安装了 ECharts 依赖
- 后端接口是否正常返回数据
- 浏览器控制台是否有报错

### Q2: 月份切换无效？

**A**: 检查：
- 日期选择器的 `value-format` 是否正确
- `@change` 事件是否正确绑定
- 后端接口参数是否正确传递

### Q3: 统计数据不准确？

**A**: 确认：
- 后端计算逻辑是否正确（支出 - 收入）
- 是否只统计了计入收支的账单
- 前端计算逻辑是否正确
