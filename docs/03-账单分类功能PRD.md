## 一、功能概述

### 1.1 功能描述

账单分类功能用于管理账单的收入和支出分类，支持两级分类结构，用户可以自定义分类并设置启用状态。

### 1.2 业务价值

- 规范分类：统一管理收支分类，便于数据统计
- 灵活组织：支持两级分类，满足细分需求
- 可控管理：支持启用/禁用，灵活控制分类显示
- 个性化：支持自定义图标和排序

### 1.3 用户角色

- 普通用户：管理分类、使用分类记账

## 二、数据库设计

### 2.1 bill_category（分类表）

#### 表说明
管理账单的收入和支出分类

#### 字段定义

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | 是 | 自增 | 主键 ID |
| category_name | VARCHAR | 50 | 是 | - | 分类名称 |
| category_type | VARCHAR | 20 | 是 | - | 分类类型：INCOME-收入分类，EXPENSE-支出分类 |
| parent_id | BIGINT | - | 否 | NULL | 父分类 ID（二级分类使用） |
| icon | VARCHAR | 50 | 否 | NULL | 图标 |
| enabled | VARCHAR | 1 | 是 | '1' | 是否启用：0-禁用，1-启用 |
| sort_order | INT | - | 是 | 0 | 排序序号 |
| creator_code | VARCHAR | 50 | 否 | NULL | 创建人编码 |
| updater_code | VARCHAR | 50 | 否 | NULL | 更新人编码 |
| creator_name | VARCHAR | 50 | 否 | NULL | 创建人姓名 |
| updater_name | VARCHAR | 50 | 否 | NULL | 更新人姓名 |
| create_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | - | 是 | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| deleted | VARCHAR | 1 | 是 | '0' | 逻辑删除标识 |

#### 分类类型枚举

| 类型值 | 说明 |
|--------|------|
| INCOME | 收入分类 |
| EXPENSE | 支出分类 |

#### 分类结构

**支持两级分类**：
- 一级分类：parent_id 为 NULL
- 二级分类：parent_id 为一级分类的 id

## 三、功能需求

### 3.1 分类管理

#### 需求描述
用户可以创建、编辑、删除、启用/禁用分类

#### 功能要点

**创建分类**：
- 输入分类名称（必填，不超过 50 字符）
- 选择分类类型（收入/支出）
- 选择父分类（可选，创建二级分类时选择）
- 选择图标（可选）
- 分类名称在同类型下不能重复

**编辑分类**：
- 修改分类名称
- 修改图标
- 调整排序
- 不能修改分类类型和父分类

**删除分类**：
- 删除前检查是否有账单使用
- 如果有账单使用，不允许删除，提示用户先禁用
- 如果有子分类，不允许删除
- 逻辑删除

**启用/禁用分类**：
- 禁用后的分类不在前端显示
- 已使用该分类的账单仍保留分类信息
- 可以重新启用
- 禁用父分类时，子分类也自动禁用

#### 预设分类

**支出分类**：
- 餐饮（早餐、午餐、晚餐）
- 交通（公交、地铁、打车）
- 购物
- 娱乐
- 住房
- 医疗
- 教育
- 其他支出

**收入分类**：
- 工资
- 奖金
- 投资收益
- 其他收入

### 3.2 分类选择

#### 需求描述
用户在记账时选择分类

#### 功能要点
- 按分类类型展示（收入/支出）
- 仅显示启用的分类
- 支持搜索分类
- 一级分类和二级分类分层展示
- 常用分类置顶显示

### 3.3 分类统计

#### 需求描述
用户可以查看各分类的收支统计

#### 统计内容
- 分类使用次数
- 分类下的总金额
- 分类占比
- 分类趋势

#### 展示方式
- 饼图：分类占比
- 柱状图：分类对比
- 列表：分类明细

## 五、接口设计

### 5.1 创建分类

**接口路径**：`POST /category`

**请求参数**：
```json
{
  "categoryName": "餐饮",
  "categoryType": "EXPENSE",
  "parentId": null,
  "icon": "icon-food"
}
```

### 5.2 查询分类列表

**接口路径**：`POST /category/list`

**请求参数**：
```json
{
  "categoryType": "EXPENSE",
  "enabled": "1"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "categoryName": "餐饮",
      "categoryType": "EXPENSE",
      "parentId": null,
      "icon": "icon-food",
      "enabled": "1",
      "sortOrder": 1,
      "children": [
        {
          "id": 2,
          "categoryName": "早餐",
          "categoryType": "EXPENSE",
          "parentId": 1,
          "enabled": "1",
          "sortOrder": 1
        }
      ]
    }
  ]
}
```

### 5.3 查询分类树

**接口路径**：`GET /category/tree`

**请求参数**：
```json
{
  "categoryType": "EXPENSE"
}
```

### 5.4 更新分类

**接口路径**：`POST /category/update`

**请求参数**：
```json
{
  "id": 1,
  "categoryName": "餐饮",
  "icon": "icon-food",
  "sortOrder": 1
}
```

### 5.5 删除分类

**接口路径**：`POST /category/delete`

**请求参数**：
```json
{
  "id": 1
}
```

### 5.6 启用/禁用分类

**接口路径**：`POST /category/toggle`

**请求参数**：
```json
{
  "id": 1,
  "enabled": "0"
}
```

### 5.7 分类统计

**接口路径**：`POST /category/statistics`

**请求参数**：
```json
{
  "categoryType": "EXPENSE",
  "startDate": "2025-01-01",
  "endDate": "2025-01-31"
}
```

## 六、业务规则

### 6.1 分类命名规则

- 分类名称不能为空
- 分类名称长度：1-50 字符
- 同类型下分类名称不能重复
- 分类名称不能包含特殊字符：`< > & " '`

### 6.2 分类层级规则

- 最多支持两级分类
- 一级分类：parent_id 为 NULL
- 二级分类：parent_id 为一级分类的 id
- 不能在二级分类下再创建子分类

### 6.3 分类删除规则

- 删除前检查是否有账单使用
- 如果有账单使用，不允许删除，提示：「该分类已被 X 个账单使用，请先禁用」
- 如果有子分类，不允许删除，提示：「该分类下有子分类，请先删除子分类」
- 删除后不可恢复（逻辑删除）

### 6.4 分类启用/禁用规则

- 禁用父分类时，子分类也自动禁用
- 启用子分类时，父分类必须是启用状态
- 禁用后的分类不在前端显示
- 已使用该分类的账单仍保留分类信息

### 6.5 分类数量限制

- 一级分类最多 50 个
- 每个一级分类下最多 20 个二级分类

## 七、非功能需求

### 7.1 性能要求

- 分类列表查询响应时间：< 500 毫秒
- 分类树查询响应时间：< 800 毫秒
- 创建分类响应时间：< 300 毫秒

### 7.2 数据安全

- 预设分类不允许删除
- 分类删除使用逻辑删除
- 记录创建人和更新人信息

### 7.3 用户体验

- 分类选择器支持搜索
- 常用分类置顶显示
- 分类图标醒目易识别
- 分类排序支持拖拽
