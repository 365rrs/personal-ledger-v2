## 🎯 功能概述

### 核心逻辑
从导入账单数据中提取关键字（配置的字段值） → 匹配清洗规则 → 完成对应字段的数据清洗

### 清洗流程
```
导入账单数据
  ↓
提取关键字段值（payment_channel、category、transaction_type 等）
  ↓
根据关键字匹配清洗规则（match_fields）
  ↓
返回清洗后的目标值（target_value）
  ↓
更新账单字段
```

## 🔧 后端实现详情

### 1. API 接口列表

| 接口 | 方法 | 路径 | 说明 |
|-----|------|------|------|
| 分页查询 | POST | `/bill-data-clean-rule/page` | 查询清洗规则列表 |
| 查询详情 | GET | `/bill-data-clean-rule/detail?id={id}` | 查询单条规则详情 |
| 创建规则 | POST | `/bill-data-clean-rule/create` | 新建清洗规则 |

| 删除规则 | POST | `/bill-data-clean-rule/delete?id={id}` | 删除清洗规则（逻辑删除） |

### 2. 规则类型枚举

| 类型值 | 说明 | 可配置字段 |
|--------|------|-----------|
| `PAYMENT_CHANNEL` | 支付渠道清洗 | payment_channel |
| `CATEGORY` | 分类清洗 | category, transaction_type |
| `TRANSACTION_DESC` | 备注清洗 | transaction_desc |

### 3. 核心 Service 方法

#### cleanField - 清洗字段
```java
String cleanField(String ruleType, Map<String, String> billFields)
```
- 按优先级匹配规则
- 支持 JSON 多字段匹配
- 返回清洗后的目标值

#### pageRules - 分页查询
```java
IPage<BillDataCleanRuleVO> pageRules(BillDataCleanRuleQueryDTO dto)
```
- 支持规则类型筛选
- 支持启用状态筛选
- 支持关键字搜索（目标值/备注）
