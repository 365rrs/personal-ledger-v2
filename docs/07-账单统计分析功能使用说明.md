# 账单统计分析功能使用说明

## 功能概述

账单统计分析功能用于统计每日的结余情况，并以折线图的形式展示。结余展示为正数（支出-收入），时间范围默认为本月，支持按月切换。

## 接口说明

### 查询每日结余

**接口地址**：`POST /bill/dailyBalance`

**请求参数**：

```json
{
  "year": 2026,
  "month": 1
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| year | Integer | 否 | 年份，不传则默认当前年份 |
| month | Integer | 否 | 月份（1-12），不传则默认当前月份 |

**响应示例**：

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "date": "2026-01-01",
      "balance": 150.50
    },
    {
      "date": "2026-01-02",
      "balance": -200.00
    },
    {
      "date": "2026-01-03",
      "balance": 80.30
    }
  ],
  "timestamp": "2026-01-13T10:30:00"
}
```

**响应字段说明**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| date | String | 日期（格式：yyyy-MM-dd） |
| balance | BigDecimal | 结余（正数表示盈余，负数表示亏损） |

## 计算规则

### 结余计算公式

```
结余 = 支出金额 - 收入金额
```

- **正数**：表示当日支出大于收入（亏损）
- **负数**：表示当日收入大于支出（盈余）
- **零**：表示当日收支平衡

### 统计范围

- 只统计 `include_in_statistics = '1'` 的账单（计入收支统计）
- 只统计 `deleted = '0'` 的账单（未删除）
- 按交易日期（transaction_date）分组统计

## 前端展示建议

### 折线图配置

使用 ECharts 或其他图表库展示折线图，建议配置如下：

```javascript
{
  xAxis: {
    type: 'category',
    data: ['2026-01-01', '2026-01-02', '2026-01-03', ...]  // 日期列表
  },
  yAxis: {
    type: 'value',
    name: '结余（元）'
  },
  series: [{
    name: '每日结余',
    type: 'line',
    data: [150.50, -200.00, 80.30, ...]  // 结余列表
  }]
}
```

### 月份切换

提供年月选择器，支持用户切换查询月份：

```vue
<el-date-picker
  v-model="selectedMonth"
  type="month"
  placeholder="选择月份"
  @change="loadDailyBalance"
/>
```

### 数据展示优化

1. **颜色区分**：
   - 正数（盈余）：使用绿色
   - 负数（亏损）：使用红色

2. **趋势分析**：
   - 显示月度总结余
   - 显示平均每日结余
   - 标注最大盈余日和最大亏损日

3. **空数据处理**：
   - 如果某日没有账单，不显示该日期的数据点
   - 提示用户"本月暂无账单数据"

## 使用示例

### 查询当前月份

```bash
curl -X POST http://localhost:8080/bill/dailyBalance \
  -H "Content-Type: application/json" \
  -d '{}'
```

### 查询指定月份

```bash
curl -X POST http://localhost:8080/bill/dailyBalance \
  -H "Content-Type: application/json" \
  -d '{
    "year": 2026,
    "month": 1
  }'
```

## 注意事项

1. **结余为正数**：按照需求，结余展示为正数（支出-收入），与常规理解相反
2. **只统计计入收支的账单**：不计入统计的账单（如转账、还款等）不会影响结余计算
3. **按日期分组**：同一天的多笔账单会合并计算结余
4. **时间范围限制**：建议只查询单个月份的数据，避免数据量过大

## 扩展功能建议

1. **支持自定义时间范围**：除了按月查询，还可以支持按周、按季度、按年查询
2. **多维度统计**：支持按分类、按支付渠道等维度统计结余
3. **趋势预测**：基于历史数据预测未来结余趋势
4. **导出报表**：支持将统计数据导出为 Excel 或 PDF 格式
