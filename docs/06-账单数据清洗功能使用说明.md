# 账单数据清洗功能使用说明

## 功能概述

账单数据清洗功能用于在导入账单时，根据预设规则自动填充或修正账单字段，提高数据质量和一致性。

## 支持的清洗类型

### 1. 分类清洗（CATEGORY）- 必填

**用途**：根据账单的原始信息自动匹配分类

**可匹配的字段**：
- `category` - 分类名称
- `sub_category` - 二级分类名称
- `amount_type` - 收支类型（INCOME/EXPENSE）
- `transaction_type` - 交易类型
- `payment_channel` - 支付渠道
- `manual_remark` - 用户备注

**示例规则**：
```json
{
  "ruleType": "CATEGORY",
  "fields": [
    {
      "name": "category",
      "matchMode": "exact",
      "value": "餐饮"
    },
    {
      "name": "amount_type",
      "matchMode": "exact",
      "value": "EXPENSE"
    }
  ],
  "targetValue": "饮食用餐",
  "priority": 100
}
```

**效果**：当账单的分类为"餐饮"且收支类型为支出时，自动设置分类为"饮食用餐"

### 2. 支付渠道清洗（PAYMENT_CHANNEL）- 可选

**用途**：统一支付渠道名称

**可匹配的字段**：
- `payment_channel` - 支付渠道
- `amount_type` - 收支类型
- `transaction_desc` - 交易描述

**示例规则**：
```json
{
  "ruleType": "PAYMENT_CHANNEL",
  "fields": [
    {
      "name": "payment_channel",
      "matchMode": "contains",
      "value": "微信"
    }
  ],
  "targetValue": "微信支付",
  "priority": 100
}
```

**效果**：当账单的支付渠道包含"微信"时，自动设置为"微信支付"

### 3. 用户备注清洗（MANUAL_REMARK）- 可选

**用途**：自动添加或修正用户备注

**可匹配的字段**：
- `manual_remark` - 用户备注
- `transaction_desc` - 交易描述
- `category` - 分类名称

**示例规则**：
```json
{
  "ruleType": "MANUAL_REMARK",
  "fields": [
    {
      "name": "category",
      "matchMode": "exact",
      "value": "外卖"
    }
  ],
  "targetValue": "饿了么订单",
  "priority": 100
}
```

**效果**：当账单分类为"外卖"时，自动添加备注"饿了么订单"

## 匹配模式说明

### 1. 精确匹配（exact）
- 完全相等才匹配
- 示例：`"微信"` 只匹配 `"微信"`，不匹配 `"微信支付"`

### 2. 包含匹配（contains）
- 包含关键词即匹配
- 示例：`"微信"` 可以匹配 `"微信支付"`、`"支付宝微信转账"`

### 3. 正则匹配（regex）
- 使用正则表达式匹配
- 示例：`"^美团.*"` 匹配所有以"美团"开头的文本

## 优先级规则

- 数字越大，优先级越高
- 多个规则按优先级从高到低依次匹配
- 一旦匹配成功，立即返回目标值
- 如果没有匹配到任何规则，保持原值或使用默认值

## 数据清洗流程

```
导入账单数据
    ↓
转换为 Bill 实体
    ↓
提取 Bill 字段到 Map
{
  "amountType": "EXPENSE",
  "paymentChannel": "微信",
  "category": "餐饮",
  "manualRemark": ""
}
    ↓
应用清洗规则（按优先级）
1. CATEGORY 规则 → "饮食用餐"
2. PAYMENT_CHANNEL 规则 → "微信支付"
3. MANUAL_REMARK 规则 → "饿了么订单"
    ↓
更新 Bill 实体字段
    ↓
保存到数据库
```

## 完整示例

### 场景：美团外卖订单导入

**原始数据**：
```json
{
  "transactionDate": "2026-03-08",
  "amount": 35.80,
  "amountType": "EXPENSE",
  "paymentChannel": "美团支付",
  "category": "餐饮",
  "transactionDesc": "美团外卖订单"
}
```

**预设清洗规则**：

规则 1 - 支付渠道清洗：
```json
{
  "ruleType": "PAYMENT_CHANNEL",
  "fields": [
    {
      "name": "paymentChannel",
      "matchMode": "contains",
      "value": "美团"
    }
  ],
  "targetValue": "美团支付",
  "priority": 100
}
```

规则 2 - 分类清洗：
```json
{
  "ruleType": "CATEGORY",
  "fields": [
    {
      "name": "category",
      "matchMode": "exact",
      "value": "餐饮"
    },
    {
      "name": "transactionDesc",
      "matchMode": "contains",
      "value": "外卖"
    }
  ],
  "targetValue": "饮食用餐",
  "priority": 90
}
```

规则 3 - 用户备注清洗：
```json
{
  "ruleType": "MANUAL_REMARK",
  "fields": [
    {
      "name": "transactionDesc",
      "matchMode": "contains",
      "value": "外卖"
    }
  ],
  "targetValue": "美团外卖订单",
  "priority": 80
}
```

**清洗后结果**：
```json
{
  "transactionDate": "2026-03-08",
  "amount": 35.80,
  "amountType": "EXPENSE",
  "paymentChannel": "美团支付",        // ← 被清洗
  "category": "饮食用餐",              // ← 被清洗
  "manualRemark": "美团外卖订单",      // ← 被清洗
  "transactionDesc": "美团外卖订单"
}
```

## 注意事项

1. **分类清洗是必填的** - 如果没有任何规则匹配，会使用默认分类（其他收入/其他支出）
2. **支付渠道和用户备注是可选的** - 如果没有匹配到规则，保持原值不变
3. **规则匹配是有顺序的** - 按优先级从高到低依次尝试
4. **支持多种匹配模式** - 根据实际情况选择合适的匹配模式
5. **字段名使用驼峰命名** - 与 Bill 实体类保持一致

## 最佳实践

1. **先宽泛后精确** - 先用包含匹配处理大类，再用精确匹配处理特殊情况
2. **合理设置优先级** - 特殊规则的优先级应高于通用规则
3. **定期清理无效规则** - 删除长期未匹配到数据的规则
4. **使用正则要谨慎** - 正则表达式性能较低，仅在必要时使用
5. **测试规则效果** - 创建规则后，先小批量测试再全面应用
