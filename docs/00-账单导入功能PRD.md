## 一、功能概述

### 1.1 功能描述

账单导入功能允许用户通过上传 Excel/CSV 文件批量导入账单数据，系统自动解析文件内容、检测重复记录、并支持用户确认后转换为正式账单。

### 1.2 业务价值

- 提高数据录入效率，支持批量导入
- 自动检测重复数据，避免账单重复
- 保留导入历史记录，支持数据追溯
- 提供导入失败原因，便于问题排查

### 1.3 用户角色

- 普通用户：上传账单文件、查看导入记录、确认转换账单

## 二、功能流程

### 2.1 整体流程图

```
用户上传文件
    ↓
文件解析与校验
    ↓
保存导入记录（bill_import_record）
    ↓
逐行解析并保存明细（bill_import_detail）
    ↓
计算数据指纹（data_hash）
    ↓
重复检测
    ↓
用户查看导入结果
    ↓
用户确认转换
    ↓
生成正式账单（bill）
```

### 2.2 详细流程说明

**设计理念**：采用分步操作设计，每个阶段独立处理，通过状态字段管理流程，便于问题排查和部分重试。

#### 阶段一：文件上传与解析

1. 用户选择 Excel/CSV 文件上传
2. 系统校验文件格式和大小
3. 创建导入记录（bill_import_record），状态为 PROCESSING
4. 解析文件内容，逐行读取数据

#### 阶段二：数据导入

1. 逐行解析数据，提取字段：类型、金额、分类、描述、交易时间
2. 计算数据指纹：`data_hash = MD5(type + amount + transaction_time + description)`
3. 批量插入导入明细表（bill_import_detail）
4. 记录导入状态：SUCCESS（成功）/ FAILED（失败）
5. 更新导入记录统计数据

**状态管理**：import_status 标记每条记录的导入结果

#### 阶段三：重复检测

1. 批量查询：根据 data_hash 在 bill 表中查找相同记录
2. 如果找到重复：
   - duplicate_status = DUPLICATE
   - duplicate_ledger_id = 重复的账单ID
   - convert_status = DUPLICATE（自动跳过）
3. 如果唯一：
   - duplicate_status = UNIQUE
   - convert_status = PENDING（待转换）

**状态管理**：duplicate_status 标记重复检测结果，convert_status 控制是否可转换

#### 阶段四：用户确认转换

1. 用户查看导入结果，包括：
   - 总记录数、成功数、失败数
   - 唯一记录数、重复记录数
   - 明细列表（可筛选状态）
2. 用户可以：
   - 查看重复记录详情
   - 手动跳过某些记录
   - 确认转换唯一记录
3. 系统批量转换账单：
   - 仅转换 duplicate_status = UNIQUE 且 convert_status = PENDING 的记录
   - 批量创建正式账单（bill）
   - 更新 convert_status = CONVERTED
   - 记录 ledger_id

**状态管理**：convert_status 标记转换结果，支持部分重试

**分步操作优势**：
- 每个阶段独立处理，中间失败不影响已完成的步骤
- 通过状态字段记录每步结果，便于问题排查
- 支持部分重试，无需重新处理整个文件
- 用户可以在任意阶段介入，手动处理异常情况

## 三、数据库设计

### 3.1 表关系

```
bill_import_record (1) ←→ (N) bill_import_detail
bill_import_detail (1) ←→ (1) bill
```

### 3.2 bill_import_record（导入记录表）

#### 表说明
记录每次批量导入的文件信息、导入状态和统计数据

#### 字段定义

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |

| total_count | INT | - | 是 | 0 | 总记录数 |
| processed_count | INT | - | 是 | 0 | 已处理记录数 |
| success_count | INT | - | 是 | 0 | 成功导入数 |
| fail_count | INT | - | 是 | 0 | 失败记录数 |
| status | VARCHAR | 20 | 是 | - | 导入状态 |
| error_message | TEXT | - | 否 | NULL | 错误信息 |
| file_path | VARCHAR | 500 | 否 | NULL | 文件存储路径 |
| start_time | DATETIME | - | 是 | - | 开始时间 |
| end_time | DATETIME | - | 否 | NULL | 结束时间 |
| creator_code | VARCHAR | 50 | 否 | NULL | 创建人编码 |

## 四、功能需求

### 4.1 文件上传

#### 需求描述
用户可以上传 Excel 或 CSV 格式的账单文件

#### 功能要点
- 支持文件格式：.xlsx、.xls、.csv
- 文件大小限制：不超过 10MB
- 文件内容校验：必须包含必填字段

#### 文件格式要求

| 列名 | 必填 | 类型 | 说明 | 示例 |
|------|------|------|------|------|
| 类型 | 是 | 文本 | INCOME 或 EXPENSE | INCOME |
| 金额 | 是 | 数字 | 大于 0 | 1000.00 |
| 描述 | 否 | 文本 | 账单描述 | 2025年1月工资 |
| 交易时间 | 是 | 日期时间 | yyyy-MM-dd HH:mm:ss | 2025-01-13 10:00:00 |

### 4.2 导入进度查询

#### 需求描述
用户可以实时查看文件导入的进度

#### 功能要点
- 显示导入进度百分比
- 显示已处理记录数 / 总记录数
- 自动刷新进度（每 2 秒轮询一次）
- 导入完成后停止轮询

#### 进度计算
```
progress = (processed_count / total_count) * 100  // 后端计算返回，不存储到数据库
```

### 4.3 导入记录列表

#### 需求描述
用户可以查看所有导入记录的历史

#### 功能要点
- 列表展示：文件名、导入时间、状态、总记录数、成功数、失败数
- 支持筛选：按状态筛选
- 支持排序：按导入时间排序
- 支持分页：每页 10 条

#### 操作按钮
- 查看详情：查看导入明细
- 删除记录：删除导入记录（逻辑删除）

### 4.4 导入明细查看

#### 需求描述
用户可以查看某次导入的所有明细记录

#### 功能要点
- 展示字段：ID、类型、金额、描述、交易时间、导入状态、重复状态、转换状态、错误信息、转换错误信息
- 支持筛选：
  - 按导入状态筛选（成功/失败）
  - 按重复状态筛选（唯一/重复）
  - 按转换状态筛选（待转换/已转换/已跳过/重复跳过/转换失败）
- 支持分页：每页 20 条

#### 状态标识
- 导入成功：绿色标签
- 导入失败：红色标签，显示错误信息
- 唯一记录：蓝色标签
- 重复记录：橙色标签，显示重复的账单ID
- 待转换：灰色标签
- 已转换：绿色标签
- 已跳过：黄色标签
- 转换失败：红色标签，显示错误信息

### 4.5 重复检测

#### 需求描述
系统自动检测导入数据是否与已有账单重复

#### 检测规则
- 计算数据指纹：`MD5(type + amount + transaction_time + description)`
- 在 bill 表中查找相同 data_hash 的记录
- 如果存在且未被删除，则标记为重复

#### 批量检测优化
- 批量插入导入明细后，统一进行重复检测
- 使用 IN 查询批量匹配 data_hash
```sql
SELECT data_hash, id FROM bill 
WHERE data_hash IN (?, ?, ...) AND deleted = '0'
```

#### 重复处理
- 自动标记：duplicate_status = DUPLICATE
- 自动跳过：convert_status = DUPLICATE
- 记录重复账单ID：duplicate_ledger_id

### 4.5 转换为账单

#### 需求描述
用户确认后，将导入的唯一记录转换为正式账单

#### 功能要点
- 仅转换条件：
  - import_status = SUCCESS
  - duplicate_status = UNIQUE
  - convert_status = PENDING
- 批量转换：支持一键转换所有符合条件的记录
- 选择转换：支持勾选部分记录转换
- 转换后更新：
  - convert_status = CONVERTED
  - ledger_id = 新创建的账单ID

#### 操作按钮
- 一键转换：转换所有待转换记录
- 批量转换：转换勾选的记录
- 跳过记录：将记录标记为 SKIPPED

## 五、非功能需求

### 5.1 性能要求

- 文件解析速度：1000 条记录 < 5 秒
- 重复检测速度：1000 条记录 < 10 秒
- 转换速度：1000 条记录 < 15 秒
- 列表查询响应时间：< 1 秒

### 5.2 数据安全

- 上传文件存储：保存到服务器指定目录
- 文件保留时间：30 天后自动清理
- 数据加密：敏感字段加密存储
- 权限控制：仅本人可查看自己的导入记录

### 5.3 异常处理

- 文件格式错误：提示用户文件格式不正确
- 文件内容错误：记录错误行号和错误原因
- 导入失败：保留失败记录，支持重新导入
- 系统异常：记录错误日志，提示用户稍后重试

## 七、接口设计

### 7.1 上传文件

**接口路径**：`POST /import/upload`

**请求参数**：
- file：文件（multipart/form-data）

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "importRecordId": 1,
    "fileName": "账单2025-01.xlsx",
    "totalCount": 100,
    "successCount": 95,
    "failCount": 5
  }
}
```

### 7.2 查询导入记录列表

**接口路径**：`POST /import/record/list`

**请求参数**：
```json
{
  "current": 1,
  "size": 10,
  "status": "SUCCESS"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [...],
    "total": 100,
    "current": 1,
    "size": 10
  }
}
```

### 7.3 查询导入明细

**接口路径**：`POST /import/detail/list`

**请求参数**：
```json
{
  "importRecordId": 1,
  "current": 1,
  "size": 20,
  "importStatus": "SUCCESS",
  "duplicateStatus": "UNIQUE",
  "convertStatus": "PENDING"
}
```

### 7.4 转换为账单

**接口路径**：`POST /import/convert`

**请求参数**：
```json
{
  "importRecordId": 1,
  "detailIds": [1, 2, 3]
}
```

### 7.5 跳过记录

**接口路径**：`POST /import/skip`

**请求参数**：
```json
{
  "detailIds": [4, 5, 6]
}
```
