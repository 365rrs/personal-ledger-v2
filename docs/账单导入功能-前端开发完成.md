## 🎨 UI设计特点

### 1. 状态标签颜色
- **导入状态**
  - 成功：绿色（success）
  - 失败：红色（danger）
  
- **重复状态**
  - 唯一：蓝色（primary）
  - 重复：橙色（warning）
  - 未检查：灰色（info）
  
- **转换状态**
  - 待转换：灰色（info）
  - 已转换：绿色（success）
  - 已跳过：橙色（warning）
  - 失败：红色（danger）

### 2. 交互设计
- 文件上传：拖拽上传 + 点击上传
- 批量操作：复选框选择 + 批量按钮
- 二次确认：删除、转换、跳过操作都有确认提示
- 消息提示：成功/失败/警告消息提示

### 3. 响应式布局
- 表格列宽自适应
- 分页组件右对齐
- 卡片头部左右布局

## 📡 API接口调用

### billImport.js 接口列表

| 函数名 | 接口路径 | 方法 | 说明 |
|--------|---------|------|------|
| uploadFile | /bill-import/upload | POST | 上传文件 |
| pageRecords | /bill-import/record/page | POST | 查询记录列表 |
| getRecord | /bill-import/record | GET | 查询记录详情 |
| pageDetails | /bill-import/detail/page | POST | 查询明细列表 |
| getStatistics | /bill-import/statistics | GET | 查询统计信息 |
| convertToBill | /bill-import/convert | POST | 转换为账单 |
| skipRecords | /bill-import/skip | POST | 跳过记录 |
| deleteRecord | /bill-import/record/delete | POST | 删除记录 |

## 🚀 使用说明

### 1. 安装依赖
```bash
cd personal-ledger-frontend
npm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

### 3. 访问页面
```
http://localhost:3000/bill-import
```

### 4. 操作流程
1. 点击"上传文件"按钮
2. 选择或拖拽Excel/CSV文件
3. 点击"开始导入"
4. 自动跳转到明细页查看导入结果
5. 筛选待转换的记录
6. 勾选记录后点击"批量转换"
7. 确认后转换为正式账单

## 📝 文件格式要求

### Excel/CSV 表头（必须完全匹配）
| 列名 | 必填 | 格式 | 示例 |
|------|------|------|------|
| 类型 | 是 | INCOME/EXPENSE | INCOME |
| 金额 | 是 | 数字 | 1000.00 |
| 分类 | 是 | 文本 | 工资 |
| 描述 | 否 | 文本 | 2025年1月工资 |
| 交易时间 | 是 | yyyy-MM-dd HH:mm:ss | 2025-01-13 10:00:00 |

### 示例数据
```csv
类型,金额,分类,描述,交易时间
INCOME,5000.00,工资,2025年1月工资,2025-01-10 10:00:00
EXPENSE,100.00,餐饮,午餐,2025-01-11 12:30:00
EXPENSE,50.00,交通,打车,2025-01-11 18:00:00
```

## ⚠️ 注意事项

1. **文件格式**：仅支持 .xlsx、.xls、.csv 格式
2. **文件大小**：不超过 10MB
3. **表头匹配**：Excel/CSV 的列名必须完全匹配（包括中文）
4. **时间格式**：必须为 yyyy-MM-dd HH:mm:ss 格式
5. **交易类型**：只能是 INCOME 或 EXPENSE
6. **批量操作**：需要先勾选记录才能进行批量操作
