## 一、功能概述

### 1.1 功能描述

账单功能是个人账本系统的核心模块，用于记录、查询、统计用户的收入和支出信息，支持手工记账和导入记账两种方式。

### 1.2 业务价值

- 记录每一笔收支明细，清晰掌握资金流向
- 支持多维度查询和统计，了解消费习惯
- 区分手工记账和导入数据，便于数据管理
- 支持标签分类，灵活组织账单数据

### 1.3 用户角色

- 普通用户：记录账单、查询账单、统计分析

## 二、数据库设计

### 2.1 bill（账单表）

#### 表说明
记录每一笔收入或支出的详细信息

#### 字段定义

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | 是 | 自增 | 主键 ID |
| transaction_date | DATE | - | 是 | - | 交易日期 |
| transaction_time | TIME | - | 否 | NULL | 交易时间 |
| income_amount | DECIMAL | 10,2 | 否 | 0.00 | 收入金额 |
| expense_amount | DECIMAL | 10,2 | 否 | 0.00 | 支出金额 |
| amount_type | VARCHAR | 20 | 是 | 'EXPENSE' | 金额类型：INCOME-收入，EXPENSE-支出 |
| transaction_type | VARCHAR | 100 | 否 | NULL | 交易类型（原始值） |
| transaction_desc | VARCHAR | 500 | 否 | NULL | 交易描述 |
| payment_channel | VARCHAR | 50 | 否 | NULL | 支付渠道 |
| payment_channel_id | BIGINT | - | 否 | NULL | 支付渠道 ID |
| category | VARCHAR | 50 | 否 | NULL | 分类 |
| category_id | BIGINT | - | 否 | NULL | 分类 ID |
| sub_category | VARCHAR | 50 | 否 | NULL | 二级分类 |
| sub_category_id | BIGINT | - | 否 | NULL | 二级分类 ID |
| manual_remark | VARCHAR | 500 | 否 | NULL | 手工备注 |
| include_in_statistics | VARCHAR | 1 | 是 | '1' | 是否计入收支统计 |
| manual_entry | VARCHAR | 1 | 是 | '0' | 是否手工记账 |
| data_hash | VARCHAR | 64 | 否 | NULL | 数据指纹 |

#### 交易类型枚举

| 类型值 | 说明 |
|--------|------|
| INCOME | 收入 |
| EXPENSE | 支出 |

#### 字段说明

**金额字段设计**：
- income_amount：收入金额，仅收入类型时有值
- expense_amount：支出金额，仅支出类型时有值
- 两个字段分开存储，便于统计和查询

**时间字段设计**：
- transaction_date：交易日期（必填），用于按日期统计
- transaction_time：交易时间（可选），记录具体时间点

**分类字段设计**：
- category：一级分类（可选），如：餐饮、交通、工资等
- category_id：一级分类 ID（可选），关联 bill_category 表
- sub_category：二级分类（可选），如：早餐、午餐、晚餐等
- sub_category_id：二级分类 ID（可选），关联 bill_category 表

**标识字段**：
- include_in_statistics：是否计入收支统计，用于排除某些特殊交易
- manual_entry：是否手工记账，区分手工录入和导入数据
- data_hash：数据指纹，用于重复检测

## 三、功能需求

### 3.1 手工记账

#### 需求描述
用户可以手动录入一笔账单

#### 功能要点
- 选择交易类型：收入/支出
- 输入金额（必填）
- 选择分类（必填）
- 选择二级分类（可选）
- 选择交易日期（必填，默认今天）
- 输入交易时间（可选）
- 输入交易描述（可选）
- 选择支付渠道（可选）
- 输入手工备注（可选）
- 选择是否计入统计（默认是）

#### 表单校验
- 金额必须大于 0
- 金额最多两位小数
- 交易日期不能晚于今天
- 分类必须选择

### 3.2 账单列表

#### 需求描述
用户可以查看所有账单记录

#### 功能要点
- 列表展示：日期、类型、金额、分类、描述、支付渠道
- 支持筛选：
  - 按交易类型筛选（收入/支出）
  - 按日期范围筛选
  - 按分类筛选
  - 按支付渠道筛选
  - 按是否手工记账筛选
- 支持排序：按日期排序（默认倒序）
- 支持分页：每页 20 条
- 显示统计：当前筛选条件下的收入总额、支出总额、结余

#### 操作按钮
- 查看详情：查看账单详细信息
- 编辑：修改账单信息
- 删除：删除账单（逻辑删除）

### 3.3 账单详情

#### 需求描述
用户可以查看账单的详细信息

#### 展示内容
- 交易日期和时间
- 交易类型
- 金额
- 分类和二级分类
- 交易描述
- 支付渠道
- 手工备注
- 是否计入统计
- 是否手工记账
- 关联的标签
- 创建时间和创建人
- 更新时间和更新人

### 3.4 编辑账单

#### 需求描述
用户可以修改账单信息

#### 功能要点
- 可修改所有字段（除了 id、创建信息）
- 修改后重新计算 data_hash
- 记录更新人和更新时间

#### 限制条件
- 导入的账单可以编辑
- 已删除的账单不能编辑

### 3.5 删除账单

#### 需求描述
用户可以删除账单

#### 功能要点
- 逻辑删除，不物理删除数据
- 删除后不计入统计
- 删除后可以恢复（管理功能）

#### 二次确认
- 删除前弹窗确认
- 显示要删除的账单信息

### 3.6 账单统计

#### 需求描述
用户可以查看收支统计数据

#### 统计维度
- 按日期统计：日、周、月、年
- 按分类统计：各分类的收入/支出占比
- 按支付渠道统计：各渠道的使用频率
- 按标签统计：各标签的收支情况

#### 统计指标
- 总收入
- 总支出
- 结余（收入-支出）
- 平均每日支出
- 最大单笔支出
- 最大单笔收入

#### 图表展示
- 折线图：收支趋势
- 饼图：分类占比
- 柱状图：月度对比

## 五、接口设计

### 5.1 创建账单

**接口路径**：`POST /bill`

**请求参数**：
```json
{
  "transactionDate": "2025-01-13",
  "transactionTime": "12:30:00",
  "transactionType": "EXPENSE",
  "expenseAmount": 100.00,
  "category": "餐饮",
  "subCategory": "午餐",
  "transactionDesc": "公司附近餐厅午餐",
  "paymentChannel": "微信支付",
  "manualRemark": "和同事一起",
  "includeInStatistics": "1"
}
```

### 5.2 查询账单列表

**接口路径**：`POST /bill/list`

**请求参数**：
```json
{
  "current": 1,
  "size": 20,
  "transactionType": "EXPENSE",
  "category": "餐饮",
  "startDate": "2025-01-01",
  "endDate": "2025-01-31",
  "manualEntry": "1"
}
```

### 5.3 查询账单详情

**接口路径**：`GET /bill`

**请求参数**：
```json
{
  "id": 1
}
```

### 5.4 更新账单

**接口路径**：`POST /bill/update`

**请求参数**：
```json
{
  "id": 1,
  "transactionDate": "2025-01-13",
  "expenseAmount": 120.00,
  "category": "餐饮"
}
```

### 5.5 删除账单

**接口路径**：`POST /bill/delete`

**请求参数**：
```json
{
  "id": 1
}
```

### 5.6 账单统计

**接口路径**：`POST /bill/statistics`

**请求参数**：
```json
{
  "startDate": "2025-01-01",
  "endDate": "2025-01-31",
  "dimension": "CATEGORY"
}
```

## 六、非功能需求

### 6.1 性能要求

- 列表查询响应时间：< 1 秒
- 创建账单响应时间：< 500 毫秒
- 统计查询响应时间：< 2 秒

### 6.2 数据安全

- 金额字段使用 DECIMAL 类型，避免精度丢失
- 所有删除操作使用逻辑删除
- 记录创建人和更新人信息

### 6.3 用户体验

- 表单输入支持快捷键
- 常用分类和渠道置顶显示
- 金额输入支持计算器功能
- 支持快速复制上一笔账单
