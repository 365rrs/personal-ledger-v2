## 一、功能概述

### 1.1 功能描述

账单标签功能允许用户为账单打上自定义标签，实现更灵活的账单分类和管理，支持一个账单关联多个标签。

### 1.2 业务价值

- 灵活分类：突破固定分类限制，自由组织账单
- 多维度查询：通过标签快速筛选账单
- 个性化管理：根据个人习惯创建标签体系
- 数据分析：按标签统计收支情况

### 1.3 用户角色

- 普通用户：创建标签、管理标签、给账单打标签

## 二、数据库设计

### 2.1 bill_tag（标签表）

#### 表说明
管理账单标签

#### 字段定义

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | 是 | 自增 | 主键 ID |
| tag_name | VARCHAR | 50 | 是 | - | 标签名称 |
| tag_category | VARCHAR | 50 | 否 | NULL | 标签分类 |
| tag_color | VARCHAR | 20 | 否 | NULL | 标签颜色 |
| sort_order | INT | - | 是 | 0 | 排序序号 |
| tag_status | VARCHAR | 10 | 是 | 'enable' | 状态：enable-启用，disable-停用 |
| create_time | DATETIME | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | - | 是 | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

### 2.2 bill_tag_relation（账单标签关联表）

#### 表说明
账单与标签的多对多关联关系表

#### 字段定义

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | 是 | 自增 | 主键ID |
| bill_id | BIGINT | - | 是 | - | 账单ID |
| tag_id | BIGINT | - | 是 | - | 标签ID |

#### 表关系

```
bill (1) ←→ (N) bill_tag_relation ←→ (N) bill_tag
```

一个账单可以有多个标签，一个标签可以关联多个账单。

## 三、功能需求

### 3.1 标签管理

#### 需求描述
用户可以创建、编辑、删除、启用/停用标签

#### 功能要点

**创建标签**：
- 输入标签名称（必填，不超过 50 字符）
- 选择标签分类（可选）
- 选择标签颜色（可选）
- 标签名称不能重复
- 默认状态为启用

**编辑标签**：
- 修改标签名称
- 修改标签分类
- 修改标签颜色
- 调整标签排序

**启用/停用标签**：
- 在标签列表中点击启用/停用开关
- 停用的标签在标签选择器中不显示
- 停用的标签仍保留在已关联的账单上，但不可再次添加
- 停用前检查是否有账单使用，如有则提示用户
- 可以重新启用已停用的标签

**删除标签**：
- 删除标签前检查是否有账单使用
- 如果有账单使用，提示用户
- 用户确认后删除标签，同时删除关联关系

#### 预设标签
系统预设常用标签及分类：

**时间类**：
- 工作日
- 周末

**场景类**：
- 旅游
- 报销

**重要程度**：
- 重要
- 紧急

**频率类**：
- 定期支出
- 一次性支出

### 3.2 给账单打标签

#### 需求描述
用户可以为账单添加或移除标签

#### 功能要点

**添加标签**：
- 在账单详情页点击"添加标签"
- 弹出标签选择器
- 支持搜索标签
- 支持快速创建新标签
- 一个账单可以添加多个标签

**移除标签**：
- 在账单详情页点击标签的 ✕ 图标
- 确认后移除标签

**批量打标签**：
- 在账单列表页勾选多个账单
- 点击"批量打标签"
- 选择要添加的标签
- 批量添加到选中的账单

### 3.3 按标签筛选账单

#### 需求描述
用户可以通过标签筛选账单

#### 功能要点
- 在账单列表页增加标签筛选器
- 支持选择多个标签
- 多个标签之间是"或"的关系（满足任一标签即可）
- 显示当前标签下的账单数量
- 显示当前标签下的收支统计

### 3.4 标签统计

#### 需求描述
用户可以查看标签的使用统计

#### 统计内容
- 标签使用次数
- 标签下的总收入
- 标签下的总支出
- 标签下的账单数量
- 标签使用趋势

#### 展示方式
- 标签列表展示使用次数
- 标签详情页展示详细统计
- 支持按时间范围统计

## 五、接口设计

### 5.1 创建标签

**接口路径**：`POST /tag`

**请求参数**：
```json
{
  "tagName": "工作日",
  "tagCategory": "时间类",
  "tagColor": "#1890ff"
}
```

### 5.2 查询标签列表

**接口路径**：`POST /tag/list`

**请求参数**：
```json
{
  "current": 1,
  "size": 20,
  "tagName": "工作",
  "tagCategory": "时间类"
}
```

### 5.3 更新标签

**接口路径**：`POST /tag/update`

**请求参数**：
```json
{
  "id": 1,
  "tagName": "工作日",
  "tagCategory": "时间类",
  "tagColor": "#1890ff",
  "sortOrder": 1
}
```

### 5.4 删除标签

**接口路径**：`POST /tag/delete`

**请求参数**：
```json
{
  "id": 1
}
```

### 5.5 启用/停用标签

**接口路径**：`POST /tag/status/update`

**请求参数**：
```json
{
  "id": 1,
  "tagStatus": "disable"  // enable-启用，disable-停用
}
```

**返回示例**：
```json
{
  "code": "SUCCESS",
  "message": "操作成功",
  "data": {
    "id": 1,
    "tagName": "工作日",
    "tagStatus": "disable"
  }
}
```

### 5.6 给账单添加标签

**接口路径**：`POST /bill/tag/add`

**请求参数**：
```json
{
  "billId": 1,
  "tagIds": [1, 2, 3]
}
```

### 5.6 移除账单标签

**接口路径**：`POST /bill/tag/remove`

**请求参数**：
```json
{
  "billId": 1,
  "tagId": 1
}
```

### 5.7 查询账单的标签

**接口路径**：`GET /bill/tag/list`

**请求参数**：
```json
{
  "billId": 1
}
```

### 5.8 按标签查询账单

**接口路径**：`POST /bill/list-by-tag`

**请求参数**：
```json
{
  "tagIds": [1, 2],
  "current": 1,
  "size": 20
}
```

## 六、业务规则

### 6.1 标签命名规则

- 标签名称不能为空
- 标签名称长度：1-50 字符
- 标签名称不能重复
- 标签名称不能包含特殊字符：`< > & " '`

### 6.2 标签分类

**实现方式**：
- 标签分类为字符串字段，用户可自由输入
- 前端提供预设分类下拉选项，同时支持自定义输入
- 后端不做分类校验，直接存储

**预设分类**：
- 时间类：按时间维度分类（工作日、周末等）
- 场景类：按使用场景分类（旅游、报销等）
- 重要程度：按重要程度分类（重要、紧急等）
- 频率类：按发生频率分类（定期、一次性等）

**分类规则**：
- 分类名称长度：1-50 字符
- 支持用户自定义分类
- 标签可以不分类（tag_category 为 NULL）
- 无需预先创建分类，直接使用即可

### 6.3 标签颜色

支持的颜色：
- 红色：#f5222d
- 橙色：#fa8c16
- 黄色：#fadb14
- 绿色：#52c41a
- 蓝色：#1890ff
- 紫色：#722ed1
- 灰色：#8c8c8c

### 6.4 标签删除规则

- 删除标签前检查是否有账单使用
- 如果有账单使用，提示：「该标签已被 X 个账单使用，删除后将移除所有关联，是否继续？」
- 用户确认后：
  - 逻辑删除标签
  - 逻辑删除所有关联关系

### 6.5 标签启用/停用规则

**停用规则**：
- 停用时检查是否有账单使用该标签
- 如有账单使用，提示：「该标签已被 X 个账单使用，停用后将无法在新增账单时选择，但已关联的账单仍保留此标签，是否继续？」
- 用户确认后可以停用
- 停用的标签：
  - 在标签选择器中不显示
  - 无法添加到新的账单
  - 已关联的账单上仍保留该标签（只读展示）
  - 在标签管理列表中显示为灰色

**启用规则**：
- 可以随时启用已停用的标签
- 启用后立即可用
- 恢复在标签选择器中的显示
- 可以重新添加到账单

**默认状态**：
- 新建标签默认为启用状态
- 系统预设标签默认为启用状态

### 6.6 标签数量限制

- 单个账单最多添加 10 个标签
- 用户最多创建 100 个标签

## 七、非功能需求

### 7.1 性能要求

- 标签列表查询响应时间：< 500 毫秒
- 添加标签响应时间：< 300 毫秒
- 按标签筛选账单响应时间：< 1 秒

### 7.2 用户体验

- 标签选择器支持搜索
- 标签选择器支持快速创建
- 标签颜色醒目易识别
- 标签排序支持拖拽
- 标签分类下拉框支持选择和输入
- 分类下拉框自动记忆用户使用过的分类
