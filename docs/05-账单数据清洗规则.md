## 一、清洗概述

### 1.1 设计思路

**核心逻辑**：从导入账单数据中提取关键字（配置的字段值） → 匹配清洗规则 → 完成对应字段的数据清洗

**清洗流程**：
```
导入账单数据
  ↓
提取关键字段值（payment_channel、category、transaction_type等）
  ↓
根据关键字匹配清洗规则（match_fields）
  ↓
返回清洗后的目标值（target_value）
  ↓
更新账单字段
```

### 1.2 清洗范围

| 清洗字段 | 依赖关键字 | 清洗方式 |
|---------|-----------|----------|
| transaction_type | income_amount, expense_amount | 根据金额判断 |
| category | category, transaction_type | 规则匹配 |
| payment_channel | payment_channel | 规则匹配 |
| transaction_desc | transaction_desc | 规则匹配 |

#### 建表SQL

```sql
CREATE TABLE IF NOT EXISTS bill_data_clean_rule (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    rule_type VARCHAR(50) NOT NULL COMMENT '规则类型：PAYMENT_CHANNEL-支付渠道，CATEGORY-分类，TRANSACTION_DESC-备注',
    match_fields VARCHAR(500) NOT NULL COMMENT '匹配字段（JSON格式）',
    target_value VARCHAR(200) NOT NULL COMMENT '目标值',
    priority INT NOT NULL DEFAULT 0 COMMENT '优先级（数字越大优先级越高）',
    enabled VARCHAR(1) NOT NULL DEFAULT '1' COMMENT '是否启用：0-禁用，1-启用',
    remark VARCHAR(500) COMMENT '备注说明',
    creator_code VARCHAR(50) COMMENT '创建人编码',
    updater_code VARCHAR(50) COMMENT '更新人编码',
    creator_name VARCHAR(50) COMMENT '创建人姓名',
    updater_name VARCHAR(50) COMMENT '更新人姓名',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted VARCHAR(1) DEFAULT '0' COMMENT '逻辑删除标识：0-未删除，1-已删除',
    INDEX idx_rule_type (rule_type),
    INDEX idx_enabled (enabled)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='数据清洗规则表';
```

## 三、清洗规则

### 3.1 交易类型清洗

根据金额字段自动判断，不需要配置规则。

```java
public String cleanTransactionType(BigDecimal incomeAmount, BigDecimal expenseAmount) {
    boolean hasIncome = incomeAmount != null && incomeAmount.compareTo(BigDecimal.ZERO) > 0;
    boolean hasExpense = expenseAmount != null && expenseAmount.compareTo(BigDecimal.ZERO) > 0;
    
    if (hasIncome && expenseAmount == null) {
        return "INCOME";
    } else if (hasExpense && incomeAmount == null) {
        return "EXPENSE";
    } else {
        throw new BusinessException("无法判断交易类型");
    }
}
```

SELECT target_value FROM bill_data_clean_rule

## 四、清洗流程

```
1. 交易类型清洗（根据金额判断）
   ↓
2. 分类清洗（根据 category + transaction_type 匹配）
   ↓
3. 支付渠道清洗（根据 payment_channel 匹配）
   ↓
4. 备注清洗（格式化处理）
```

## 五、Java实现示例

```java
public String cleanField(String ruleType, Map<String, String> billFields) {
    // 查询所有规则
    List<BillDataCleanRule> rules = billDataCleanRuleMapper.selectList(
        new QueryWrapper<BillDataCleanRule>()
            .eq("rule_type", ruleType)
            .eq("enabled", "1")
            .eq("deleted", "0")
            .orderByDesc("priority")
    );
    
    // 遍历规则进行匹配
    for (BillDataCleanRule rule : rules) {
        Map<String, String> matchFields = JSON.parseObject(rule.getMatchFields(), 
            new TypeReference<Map<String, String>>() {});
        
        // 检查所有字段是否匹配
        boolean allMatch = true;
        for (Map.Entry<String, String> entry : matchFields.entrySet()) {
            String fieldName = entry.getKey();
            String fieldValue = entry.getValue();
            
            if (!fieldValue.equals(billFields.get(fieldName))) {
                allMatch = false;
                break;
            }
        }
        
        if (allMatch) {
            return rule.getTargetValue();
        }
    }
    
    return null;
}
```

### 7.10 核心代码示例

#### 匹配条件配置组件

```vue
<template>
  <div class="match-fields">
    <div v-for="(field, index) in fields" :key="index" class="field-row">
      <el-select v-model="field.name" placeholder="选择字段">
        <el-option 
          v-for="opt in fieldOptions" 
          :key="opt.value" 
          :label="opt.label" 
          :value="opt.value"
        />
      </el-select>
      <el-input v-model="field.value" placeholder="字段值" />
      <el-button icon="Delete" @click="removeField(index)" />
    </div>
    <el-button type="primary" plain @click="addField">+ 添加匹配字段</el-button>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'

const props = defineProps({
  ruleType: String,
  modelValue: Object
})

const emit = defineEmits(['update:modelValue'])

const fields = ref([])

const fieldOptions = computed(() => {
  const options = {
    PAYMENT_CHANNEL: [
      { label: '支付渠道', value: 'payment_channel' }
    ],
    CATEGORY: [
      { label: '分类', value: 'category' },
      { label: '交易类型', value: 'transaction_type' }
    ],
    TRANSACTION_DESC: [
      { label: '备注', value: 'transaction_desc' }
    ]
  }
  return options[props.ruleType] || []
})

const addField = () => {
  fields.value.push({ name: '', value: '' })
}

const removeField = (index) => {
  fields.value.splice(index, 1)
  updateValue()
}

const updateValue = () => {
  const result = {}
  fields.value.forEach(f => {
    if (f.name && f.value) {
      result[f.name] = f.value
    }
  })
  emit('update:modelValue', result)
}
</script>
```
