## 一、功能概述

### 1.1 功能描述

支付渠道功能用于管理账单的支付方式，用户可以自定义支付渠道并设置启用状态，便于统计各渠道的使用情况。

### 1.2 业务价值

- 规范管理：统一管理支付渠道，便于数据统计
- 灵活配置：支持自定义渠道，满足个性化需求
- 可控管理：支持启用/禁用，灵活控制渠道显示
- 数据分析：按渠道统计收支情况，了解支付习惯

### 1.3 用户角色

- 普通用户：管理渠道、使用渠道记账

## 二、数据库设计

### 2.1 bill_payment_channel（支付渠道表）

#### 表说明
管理账单的支付渠道

#### 字段定义

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | 是 | 自增 | 主键ID |

| 类型值 | 说明 |
|--------|------|
| CASH | 现金 |
| BANK_CARD | 银行卡 |
| CREDIT_CARD | 信用卡 |
| E_WALLET | 电子钱包 |
| OTHER | 其他 |

## 三、功能需求

### 3.1 渠道管理

#### 需求描述
用户可以创建、编辑、删除、启用/禁用支付渠道

#### 功能要点

**创建渠道**：
- 输入渠道名称（必填，不超过 50 字符）
- 选择渠道类型（可选）
- 选择图标（可选）
- 渠道名称不能重复

**编辑渠道**：
- 修改渠道名称
- 修改渠道类型
- 修改图标
- 调整排序

**删除渠道**：
- 删除前检查是否有账单使用
- 如果有账单使用，不允许删除，提示用户先禁用
- 逻辑删除

**启用/禁用渠道**：
- 禁用后的渠道不在前端显示
- 已使用该渠道的账单仍保留渠道信息
- 可以重新启用

#### 预设渠道

**现金类**：
- 现金

**银行卡类**：
- 工商银行
- 建设银行
- 农业银行
- 中国银行
- 招商银行

**信用卡类**：
- 信用卡

**电子钱包类**：
- 微信支付
- 支付宝
- 云闪付

**其他**：
- 其他渠道

### 3.2 渠道选择

#### 需求描述
用户在记账时选择支付渠道

#### 功能要点
- 按渠道类型分组展示
- 仅显示启用的渠道
- 支持搜索渠道
- 常用渠道置顶显示

### 3.3 渠道统计

#### 需求描述
用户可以查看各渠道的使用统计

#### 统计内容
- 渠道使用次数
- 渠道下的总收入
- 渠道下的总支出
- 渠道使用趋势

#### 展示方式
- 饼图：渠道占比
- 柱状图：渠道对比
- 列表：渠道明细

## 五、接口设计

### 5.1 创建渠道

**接口路径**：`POST /payment-channel`

**请求参数**：
```json
{
  "channelName": "微信支付",
  "channelType": "E_WALLET",
  "icon": "icon-wechat"
}
```

### 5.2 查询渠道列表

**接口路径**：`POST /payment-channel/list`

**请求参数**：
```json
{
  "current": 1,
  "size": 20,
  "channelName": "微信",
  "channelType": "E_WALLET",
  "enabled": "1"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [
      {
        "id": 1,
        "channelName": "微信支付",
        "channelType": "E_WALLET",
        "icon": "icon-wechat",
        "enabled": "1",
        "sortOrder": 1
      }
    ],
    "total": 10,
    "current": 1,
    "size": 20
  }
}
```

### 5.3 查询渠道详情

**接口路径**：`GET /payment-channel`

**请求参数**：
```json
{
  "id": 1
}
```

### 5.4 更新渠道

**接口路径**：`POST /payment-channel/update`

**请求参数**：
```json
{
  "id": 1,
  "channelName": "微信支付",
  "channelType": "E_WALLET",
  "icon": "icon-wechat",
  "sortOrder": 1
}
```

### 5.5 删除渠道

**接口路径**：`POST /payment-channel/delete`

**请求参数**：
```json
{
  "id": 1
}
```

### 5.6 启用/禁用渠道

**接口路径**：`POST /payment-channel/toggle`

**请求参数**：
```json
{
  "id": 1,
  "enabled": "0"
}
```

### 5.7 渠道统计

**接口路径**：`POST /payment-channel/statistics`

**请求参数**：
```json
{
  "startDate": "2025-01-01",
  "endDate": "2025-01-31"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "channelId": 1,
      "channelName": "微信支付",
      "incomeAmount": 1000.00,
      "expenseAmount": 4000.00,
      "totalCount": 120,
      "percentage": 40.0
    }
  ]

```

## 六、业务规则

### 6.1 渠道命名规则

- 渠道名称不能为空
- 渠道名称长度：1-50 字符
- 渠道名称不能重复
- 渠道名称不能包含特殊字符：`< > & " '`

### 6.2 渠道类型规则

**实现方式**：
- 渠道类型为可选字段，用户可自由选择或不选
- 前端提供预设类型下拉选项
- 后端存储枚举值

**预设类型**：
- CASH：现金
- BANK_CARD：银行卡
- CREDIT_CARD：信用卡
- E_WALLET：电子钱包
- OTHER：其他

### 6.3 渠道删除规则

- 删除前检查是否有账单使用
- 如果有账单使用，不允许删除，提示：「该渠道已被 X 个账单使用，请先禁用」
- 删除后不可恢复（逻辑删除）

### 6.4 渠道启用/禁用规则

- 禁用后的渠道不在前端显示
- 已使用该渠道的账单仍保留渠道信息
- 可以重新启用

### 6.5 渠道数量限制

- 用户最多创建 100 个支付渠道

## 七、非功能需求

### 7.1 性能要求

- 渠道列表查询响应时间：< 500 毫秒
- 创建渠道响应时间：< 300 毫秒
- 渠道统计查询响应时间：< 1 秒

### 7.2 数据安全

- 预设渠道不允许删除
- 渠道删除使用逻辑删除
- 记录创建人和更新人信息

### 7.3 用户体验

- 渠道选择器支持搜索
- 常用渠道置顶显示
- 渠道图标醒目易识别
- 渠道排序支持拖拽
- 渠道按类型分组展示
